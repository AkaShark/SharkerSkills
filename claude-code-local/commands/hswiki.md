把我们在**本次对话中整理出的结论性内容**，按话题**自动拆分**成若干篇 raw 源笔记，存进「回森架构知识库」。

## 步骤

1. **解析目标目录与日期**：运行
   `echo "DIR=${HSWIKI_RAW:-/Users/sharker/Desktop/Obisidian/Obisidian/Obsidian/LLMWiki/回森架构知识库/raw}"; date +%F`
   —— 输出里的 `DIR` 就是写入目录（优先环境变量 `$HSWIKI_RAW`，未设置则用默认路径），日期用于文件名前缀。
2. **切话题**：通读本次对话，识别出几个**互相独立、各自能成立**的知识话题（如「配音三路同步方案」「组件路由设计」「构建体系」是三个话题）。
   - **宁少勿碎**：紧密相关的并成一篇；只有一两句、不成体系的别单独成篇，并到最相关的那篇里。
   - 丢掉我提问的原话、闲聊、过程性试错——只留**沉淀价值高的结论**。
   - 若我带了参数 `$ARGUMENTS`：把它当**范围过滤**（只存与之相关的话题）或**命名提示**，其余话题忽略。留空 = 全自动拆分全部值得留的内容。
3. **先列计划**：用一个简短清单告诉我打算建几篇，每篇「标题 + 一句话主题」，让我能一眼看出拆得对不对。
4. **逐篇写入** `<DIR>/`，每篇：
   - 文件名 = `<今天日期> <你自动起的简短标题>.md`（中文、概括该话题，例：`2026-06-04 配音三路音视频同步方案.md`）。
   - 顶部 frontmatter：
     ```yaml
     ---
     date: <今天日期>
     source: hisense-ios-project（在工程目录整理）
     confidence: 待确认
     topic: <该篇的话题，一词概括，便于后续分类>
     tags: [回森, raw]
     ---
     ```
   - 正文 = 该话题整理后的结构化 markdown。
   - **同名文件已存在** → 先 Read 出来，在末尾追加 `\n\n---\n\n` 分隔再续写，**绝不覆盖**。
5. **回执**：最后给我一张表 `文件路径 | 话题 | 一句话`，并补一句「分得不对就说哪几篇合并/重拆」。

## 红线

- 回森雇主内部内容，**仅本机处理、绝不外发**（不进 web search / 外部 API / gist / 任何离开本机的渠道）。
- **不要 ingest**：本命令只把素材按话题丢进 `raw/`。ingest（建 entities/concepts、更新 synthesis）由我之后在 vault 的会话里说「ingest」时再触发。
