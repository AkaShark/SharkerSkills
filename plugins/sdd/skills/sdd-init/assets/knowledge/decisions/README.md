# decisions/ — 架构决策记录 (ADR)

每个文件记录**一个有取舍的决策**：背景、被否决的备选方案、最终选择 + 理由。

- 触发：定完一个技术/架构方案后（`sdd-design` 结尾自动沉淀）。
- 文件名：决策动词短语，如 `use-jwt-over-session.md`、`split-by-feature-not-layer.md`。
- 模板见 `../../schema.md` §3.1（type=decision）；写完回写 `../index.md`。

> 一条知识若同时是「坑」和「决策」，正文记为 lesson，这里只 `[[链接]]` 它，不复制正文。
