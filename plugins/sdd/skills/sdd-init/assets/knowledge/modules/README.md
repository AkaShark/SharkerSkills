# modules/ — 模块/组件实体页

每个文件描述**一个模块/组件**：它干什么、关键文件在哪、为什么这样设计、对外接口、已知约束。

- 触发：做完一个 feature / 重构一个模块后（`sdd-impl` 完成时自动更新）。
- 文件名：kebab-case 模块名，如 `auth-service.md`、`image-uploader.md`。
- 模板见 `../../schema.md` §3.1；每页带 front-matter（type=module），写完回写 `../index.md`。
